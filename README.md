# Proxiome Analysis Template

This repository holds the Proxiome Analysis Template for single cell PNA data. 

The template is a standalone Quarto (.qmd) file that you can use as a starting point for analysis, 
and includes analysis steps that are typically performed for an analysis of PNA data. 
However, you are free to customize it as needed for your specific dataset and research questions.

## Setup & Prerequisites

This template is designed to be executed using R and Quarto. 

### Local Environment

To run the template locally, you must have the following dependencies installed:
R 
RStudio (Recommended), Visual Studio Code, or another R-compatible IDE
Quarto CLI (optional, for rendering the .qmd file)

### Required R Packages

The analysis code relies on several packages. These can typically be installed by running the 
following commands in your R console:

```
# Install pak package manager (if not already installed)
install.packages("pak")

# Install required packages using pak
pak::pak(c(
  "tidyverse",
  "Seurat",
  "harmony",
  "ComplexHeatmap",
  "ggplotify",
  "here",
  "Matrix",
  "ggraph"
))

# Install pixelatorR package from GitHub
pak::pak("PixelgenTechnologies/pixelatorR")

```

## Input Data

The template requires two types of input: the samplesheet (.csv) and the raw PNA data files (.pxl file format) generated
from the `nf-core/pixelator` pipeline.

### Samplesheet

The analysis requires a metadata file (samplesheet) in CSV format that maps sample information to file paths. 
This file must contain the following required columns:

- **sample_id**: A short, unique identifier for each sample (e.g., `S1`, `S2`, `S3`)
- **sample_alias**: A more descriptive name for the sample (e.g., `S1_resting`, `S2_PHA`)
- **file_path**: The full path to the corresponding PNA file for each sample
- **condition**: A description of the sample condition

You can include additional columns as needed (e.g., `donor`, `time_point`, `treatment`). These will be automatically 
added to the PNA object as metadata columns and can be used for downstream analysis and visualization.

Example samplesheet:

```
sample_id,sample_alias,file_path,condition,donor
S1,S1_resting,/User/username/data/S1_resting.pna,resting,D1
S2,S2_PHA,/User/username/data/S2_PHA.pna,stimulated,D2
S3,S3_IL2,/User/username/data/S3_IL2.pna,IL2_treated,D1
```

### Folder structure

The template Quarto file assumes that you have this folder structure:

```
.
├── data/                           # Input data (optional, can be placed elsewhere)
├── results/                        # Output results (will be generated by the template)
└── proxiome_analysis_template.qmd  # SOP Quarto file (can be renamed)
```

## Expected Outputs

The rendering process will populate the results/ folder with .png and .pdf files of generated figures, as well as .rds
files with processed data.
